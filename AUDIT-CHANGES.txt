P3 — MOBILE BUGS (root-caused & fixed)
────────────────────────────────────────────────────────
1) Mobile menu opened off-screen / bunched at top.
   ROOT CAUSE: nav had `backdrop-filter: blur()`. Per the CSS spec, any
   backdrop-filter turns the element into a *containing block* for its
   position:fixed children, so the menu overlay was trapped inside the
   ~64px nav bar instead of filling the screen.
   FIX (site.css): disable the blur at <=768px so the overlay becomes
   truly viewport-fixed. Desktop glass effect is untouched.
   VERIFIED with a real mobile tap: menu now fills 844px, offsetParent=null.

2) Hero video showed a play button instead of autoplaying on mobile.
   ROOT CAUSE: the old site.js deliberately stripped the background-video
   source on mobile (poster-only).
   FIX (site.js): real mobile autoplay — forces muted + playsinline, calls
   play() with a retry on first touch/scroll, and only falls back to the
   poster if autoplay is truly refused (or Data-Saver is on).

3) (Found during verification) Breadcrumb <nav> was pinned to the top,
   overlapping the menu, because the global `nav { position:fixed }` rule
   also caught the breadcrumb element.
   FIX (site.css): `.crumbs { position: static }`. Verified no overlap.
